4. A voodoo-enabled Windows 'widget' will be done compiling shortly, and we'll post binaries and links to integrity validation tools) in the voodoo thread. Or keep an eye on the @cryptohaven twitter feed, as we'll cross-post any voodoo items there.

5. Your voodoo token is totally separate from conventional cryptostorm tokens, and the cryptostorm network itself. That's just for the alpha test, but for now they are disconnected. We suggest you fiddle with the voodoo session and try various network analytic tools whilst on-voodoo: note performance, reliability, any problems particular applications may have, and so on.

6. We'll be adding additional voodoo nodes, with dedicated resolvers, as time goes on - and already have several in production. Watch the voodoo thread for resolver mappings that can be used to test out the new voodoo trajectories as they come available.

7. Your voodoo token is actually the public key of an ECC (c25519) keypair - which is the foundation of tokens(2), our upgraded token auth system. Tokens(2) is still pre-alpha, but we're using the voodoo alpha test to gather early data. If we're ready to test other bits of tokens(2) meanwhile, we'll let you know because once that happens... your voodoo token will grow wings.
